# 🔐 Персональний Цифровий Підпис

Демонстраційна програма для вивчення принципів цифрового підпису та асиметричної криптографії.

## 📋 Опис

Програма демонструє базові концепції цифрового підпису:

- **Генерація ключів** — створення пари приватного та публічного ключів на основі персональних даних
- **Підписання документів** — створення унікального цифрового підпису для файлу
- **Верифікація** — перевірка автентичності та цілісності підписаного документа
- **Виявлення змін** — демонстрація того, як підпис виявляє будь-які модифікації

## 📖 Як це працює

### Криптографічна модель

Програма використовує спрощену асиметричну модель на основі модульної арифметики:

```
Просте число Мерсенна: p = 2¹²⁷ - 1

Приватний ключ: private = SHA256(name + birthdate + seed) mod p
Публічний ключ: public = private⁻¹ mod p  (мультиплікативна обернена)

Підпис:        signature = (hash(document) × private) mod p
Верифікація:   (signature × public) mod p == hash(document)
```

## 📁 Структура файлів

```
.
├── main.py    	          # Головний скрипт
├── README.md             # Документація
├── private.key           # Приватний ключ (генерується)
├── public.key            # Публічний ключ (генерується)
├── <contract>.txt        # Документ користувача (створюється)
└── <contract>.txt.sig    # Цифровий підпис (створюється)
```

## ⚠️ Важливо

> **Це навчальна демонстрація!**
> 
> Програма НЕ призначена для реального криптографічного захисту:
> 
> - Ключі генеруються детерміновано з відомих даних
> - Використовується спрощена математична модель
> - Відсутній захист від атак на часові канали
> - Приватний ключ зберігається у відкритому вигляді

Для реальних застосувань використовуйте перевірені бібліотеки: `cryptography`, `PyCryptodome`, `PyNaCl`.

## 📚 Навчальні цілі

Після роботи з програмою ви зрозумієте:

1. **Асиметричну криптографію** — концепцію пари ключів, де один шифрує, інший дешифрує
2. **Цифровий підпис** — як математично прив'язати документ до власника ключа
3. **Цілісність даних** — чому будь-яка зміна документа робить підпис недійсним
4. **Хеш-функції** — роль SHA-256 у створенні "відбитка" документа
