# LSB-стеганографія

Інтерактивний навчальний інструмент для демонстрації стеганографії методом LSB. Скрипт уміє вбудовувати текст у зображення, відновлювати прихований текст, а також аналізувати різницю між оригіналом і модифікованим файлом із підсвіткою змінених пікселів.

## Можливості

- **Вбудовування повідомлення** у 1–2 молодші біти кожного кольорового каналу (RGB/RGBA) з підрахунком місткості та статистикою змін.
- **Відновлення повідомлення** з того самого числа LSB; опційно використовується пароль, що шифрує дані потоком XOR на основі SHA-256.
- **Аналіз відмінностей** між двома зображеннями: кількість змінених каналів, середня/максимальна різниця, розміри файлів.
- **Візуалізація різниць** через `Pillow.ImageChops.difference` з регульованим коефіцієнтом підсилення яскравості.
- Повністю **україномовний інтерфейс** із підказками та таблицями, оформленими через `rich`.

## Вимоги

- Python 3.10+
- Зовнішні пакети: `Pillow`, `rich`

```bash
python3 -m pip install -r requirements.txt
```

Рекомендується використовувати ізольоване середовище (`python3 -m venv venv && source venv/bin/activate`).

## Запуск

```bash
python3 main.py
```

Після старту в терміналі з’явиться меню:

1. **Сховати повідомлення** – запитує шлях до вихідного та цільового зображень, джерело тексту (ввід із клавіатури або файл), пароль та кількість бітів. Після вбудовування показує статистику й за потреби зберігає diff-карту.
2. **Витягнути повідомлення** – читає текст зі стегозображення, просить пароль (якщо використовувався) та дозволяє зберегти результат у файл.
3. **Аналізувати різницю** – порівнює будь-які два зображення однакового розміру і, за бажання, будує підсилену візуалізацію.

Для завершення натисніть `Q`.

## Структура проєкту

- `main.py` – інтерактивний CLI на `rich`.
- `src/__init__.py` – ядро з функціями `hide_message`, `extract_message`, `analyze_images`, `visualize_diff` та допоміжними структурами `EmbedReport`, `DiffReport`.
- `requirements.txt` – список залежностей.

## Технічні деталі

- **Формат корисного навантаження**: магічна сигнатура `LSBMSG1`, довжина повідомлення (4 байти big-endian) та зашифровані дані (якщо задано пароль).
- **Шифрування**: XOR із безкінечним байтовим потоком, що генерується як `SHA256(password + counter)`.
- **Стримінг даних**: бітовий курсор споживає payload без проміжних списків, а читання LSB виконується генераторами, що знижує використання пам’яті.

## Перевірка

Автоматичні тести не поставляються. Для перевірки:

1. Візьміть PNG з достатнім розміром (наприклад, `123.png`).
2. Запустіть режим «Сховати повідомлення» й вбудуйте короткий текст.
3. Витягніть текст із отриманого файлу та переконайтеся, що він збігається з оригіналом.
4. Використайте режим аналізу/візуалізації, щоб побачити лише мінімальні зміни (підсилення ≥ 32 допоможе виявити контури).
